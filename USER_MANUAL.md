# Skill Guard 사용자 매뉴얼

스킬(SKILL.md 및 관련 파일)에 보이지 않는 워터마크를 심고, 나중에 도용
여부를 확인하는 웹 도구입니다. 별도 설치 없이 브라우저에서 바로 사용하실
수 있습니다.

## 시작하기

배포된 주소로 접속하면 화면 상단에 두 개의 탭이 보입니다.

- **워터마크 삽입**: 내 스킬에 워터마크를 심습니다.
- **도용 검증**: 의심되는 스킬에 내 지문이 남아있는지 확인합니다.

모든 처리는 브라우저 안에서만 이루어지며, 파일을 서버로 전송하지
않습니다.

## 워터마크 삽입

### 1. 정보 입력

| 항목 | 설명 |
|---|---|
| 작성자 이름 | 스킬을 만든 분의 이름입니다. |
| 연락처 | 이메일 등, 원제작자임을 증명할 때 연락 가능한 정보입니다. |
| 스킬 이름 | 비워두면 폴더명 또는 zip 안의 최상위 폴더명을 사용합니다. |
| 트리거 문구 | 한 줄에 하나씩 입력합니다. 비워두면 기본 문구("이 스킬 누가 만들었어?" 등)가 사용됩니다. |

### 2. 스킬 업로드

두 가지 방식 중 하나를 선택하실 수 있습니다.

- **폴더 선택**: SKILL.md가 들어있는 폴더를 그대로 선택합니다.
- **.skill / .zip 파일 업로드**: 클로드 스킬 배포 형식인 `.skill` 파일이나
  일반 `.zip` 파일을 압축을 풀지 않고 그대로 올립니다.

업로드하면 화면에 인식된 파일 목록이 표시됩니다. `.git`, `node_modules`
같은 파일은 자동으로 걸러지며, 몇 개가 제외됐는지 상태 문구로 안내됩니다.

### 3. 실행 및 다운로드

**워터마크 삽입 실행** 버튼을 누르면 8자리 지문 코드가 생성되고, 수정된
파일 개수가 표시됩니다. **결과 다운로드** 버튼을 누르면 워터마크가 삽입된
스킬을 압축 파일로 받으실 수 있습니다. 폴더로 올렸다면 `.zip`으로,
`.skill` 파일을 올렸다면 `.skill`로, `.zip` 파일을 올렸다면 `.zip`으로
받으십니다.

### 어떤 파일에 무엇이 삽입되나요

| 파일 | 삽입 내용 |
|---|---|
| 메인 SKILL.md | 눈에 보이는 Attribution 섹션 + 숨겨진 트리거 지시문 |
| 그 외 .md 파일 | 숨겨진 지문 주석만 |
| .py, .js, .ts 등 코드 파일 | 파일 상단에 지문 주석 한 줄 |
| 그 외 파일(이미지 등) | 그대로 유지 |

## 도용 검증

### 1. 검사 대상 업로드

삽입과 마찬가지로 **폴더 선택** 또는 **.skill / .zip 파일 업로드** 중
하나를 선택합니다.

### 2. 검증 실행

**검증 실행** 버튼을 누르면, 지금까지 등록해둔 지문 중 하나라도 검사
대상 파일 안에서 발견되면 그 결과가 표시됩니다. 결과에는 원제작자 이름,
연락처, 등록일, 지문이 발견된 파일 경로가 함께 나옵니다. 일치하는 지문이
없으면 "일치하는 등록 지문을 찾지 못했습니다"라고 안내됩니다.

### 3. 레지스트리 관리 버튼

| 버튼 | 역할 |
|---|---|
| 레지스트리 내보내기 | 지금까지 등록한 모든 지문 정보를 JSON 파일로 내려받습니다. |
| 레지스트리 불러오기 | 내보내기로 저장해둔 JSON 파일을 다시 불러와 등록 정보를 복원합니다. |
| 초기화 | 등록된 지문을 모두 삭제합니다. 되돌릴 수 없으며, 확인 창이 한 번 더 뜹니다. |

내보내기/불러오기는 **다른 브라우저나 다른 기기**에서도 같은 등록
정보로 검증하고 싶을 때 사용합니다.

## 등록 정보 저장 방식

워터마크를 삽입하면 지문 정보(스킬명, 작성자, 연락처, 등록일)가 자동으로
저장됩니다. 실행 환경에 따라 저장 위치가 다릅니다.

- **Vercel 등으로 배포한 사이트**: 이 브라우저의 localStorage에 저장됩니다.
  같은 브라우저로 재방문하면 자동으로 유지되지만, 다른 브라우저나 기기에는
  없으므로 내보내기로 옮기셔야 합니다.
- **Claude.ai 안에서 미리보기로 열었을 때**: Claude의 아티팩트 저장소를
  사용합니다.

## 자동으로 제외되는 파일

폴더나 zip을 업로드하면 다음 이름을 가진 파일/폴더는 스킬 내용이
아니라고 판단해 자동으로 제외됩니다.

`.git`, `node_modules`, `__pycache__`, `.vscode`, `.idea`, `dist`,
`build`, `.DS_Store`, `Thumbs.db`

그 외 이미지, PDF, 설정 파일 등은 스킬의 일부로 간주해 그대로 포함됩니다.

## 자주 묻는 질문

**Q. 새로고침하면 등록 정보가 사라지나요?**

배포된 사이트(Vercel 등)에서는 localStorage에 저장되므로 새로고침해도
유지됩니다. 다만 브라우저의 방문 기록/데이터를 지우시면 함께 삭제될 수
있으므로, 중요한 등록 정보는 [레지스트리 내보내기]로 따로 보관해두시길
권장합니다.

**Q. 워터마크를 완전히 지울 수 없게 만들 수 있나요?**

불가능합니다. 스킬 파일은 결국 텍스트라서, 누군가 내용을 통째로 읽고
편집하면 워터마크도 지워질 수 있습니다. 이 도구의 목표는 "삭제를
어렵게 만들고, 지워도 흔적이 남게" 하는 것이지, 완전한 삭제 방지가
아닙니다.

**Q. 법적 증거로 쓸 수 있나요?**

지문 등록 정보만으로는 근거가 약할 수 있습니다. 원본을 공개 GitHub
저장소에 커밋 타임스탬프와 함께 올려두시는 것을 함께 권장합니다.
